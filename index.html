    Actividad 12 — Creación de interfaces de usuario adaptables y responsivas

    Punto 2 — Diseño de la interfaz

    La página está compuesta por estructuras semánticas bien definidas: header, nav, main y footer. Cada una cumple un rol específico en el layout y se adapta al tamaño de la pantalla.

    Al hacer clic, el resto de la página se oscurece y la estructura seleccionada queda resaltada con un borde azul.

    Punto 3 — Diseño responsivo

    La interfaz usa un enfoque mobile-first con Tailwind CSS. El contenido se adapta fluidamente a cualquier tamaño de pantalla:

    • En pantallas grandes (≥640 px) el menú de navegación se muestra en horizontal con dropdowns.
    • En pantallas pequeñas (<640 px) el menú se reemplaza por un botón hamburguesa (☰) en la esquina superior derecha que despliega el menú completo verticalmente.
    • El contenido principal ocupa una sola columna en móvil y se expande con márgenes en escritorio.
    • Tipografía escalable: los títulos usan text-2xl sm:text-3xl para ajustarse al dispositivo.
    • Inputs y botones tienen ancho fluido (w-full, flex-wrap) para no desbordarse en pantallas angostas.

    Punto 4 — Diseño inclusivo y accesible

    La interfaz fue construida con accesibilidad como prioridad:

    • Navegación por teclado: todos los botones, enlaces e inputs son alcanzables con Tab y activables con Enter.
    • Etiquetas ARIA: se usan aria-label, aria-required, aria-live, aria-expanded y aria-pressed en los elementos interactivos.
    • Jerarquía semántica: el documento usa <header>, <nav>, <main>, <section> y <footer> correctamente.
    • Mensajes de error: los campos inválidos muestran texto descriptivo con role="alert" para lectores de pantalla.
    • Focus visible: todos los elementos interactivos muestran un anillo de foco (focus:ring-2) claro y contrastante.

    Punto 5 — Framework utilizado: Tailwind CSS

    Este proyecto usa Tailwind CSS como framework de diseño. Tailwind es un framework utility-first que permite construir interfaces directamente en el HTML usando clases predefinidas, sin escribir CSS personalizado para la mayoría de los casos.
